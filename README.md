# zhongnan-atlas-cd-course

> 中南标图集 × AI = 施工图前辈随叫随到

我们施工图设计课要用 6 本中南标图集。这些 PDF 都是扫描件（图片），又大又传不动。我用 OCR（电脑识字）把 6 本图集的文字全部识别出来，整理成了**轻量的文本文件**（总共才 500 KB）。你把这些文件发给任何 AI，它就能帮你查做法编号、找页码、解释构造——**相当于给 AI 装了一个"图集前辈"的大脑**。

---

## 能帮你干什么

- 问"**楼101 是什么做法**"→ 告诉你每层材料、厚度，出自第几页
- 问"**平屋面排水口节点怎么画**"→ 告诉你去哪本图集第几页看
- 问"**卫生间用什么做法编号**"→ 推荐 F / XF 系列，解释区别
- 问"**天沟防水怎么做**"→ 跨多本图集给你汇总
- 问"**5+9A+5 中空玻璃 K 值多少**"→ 从节能门窗图集查数据
- 问"**XX 术语什么意思**"→ 用大白话解释

**每个答案都会带出处**（哪本图集、第几页），你拿着页码去翻原图集核对就行。

## 怎么用

### 方法一：AI 编程助手快速安装（Kiro / Claude Code / Codex / Trae 等）

在对话里说：

```
安装 skill：https://github.com/Xarles-Lovell/zhongnan-atlas-cd-course
```

工具会自动克隆 `zhongnan-atlas-cd-course/` 文件夹到你的工作区，安装后直接问施工图问题即可。

### 方法二：直接发 GitHub 链接（最懒，适合支持联网的 AI）

如果你用的 AI 支持联网/读链接（比如 Kimi、ChatGPT 联网模式、豆包等），**不用下载任何文件**，直接在对话里发这段话：

```
请你访问这个链接 https://github.com/Xarles-Lovell/zhongnan-atlas-cd-course/blob/main/zhongnan-atlas-cd-course/SKILL.md 阅读里面的内容，按照要求扮演施工图前辈来回答我的问题。再访问 https://github.com/Xarles-Lovell/zhongnan-atlas-cd-course/tree/main/zhongnan-atlas-cd-course/references 读取里面的索引文件作为你的知识库。
```

然后直接问问题就行。

> 💡 如果 AI 说"无法访问链接"，说明它不支持联网，换下面的方法三。

### 方法三：下载文件发给 AI（通用，任何 AI 都能用）

**第一步**：下载 `zhongnan-atlas-cd-course/` 文件夹

**第二步**：在你习惯用的 AI 里开一个新对话，把 `SKILL.md` 的内容复制粘贴进去（或上传），跟 AI 说：

> 请你阅读以下内容，按照里面的要求扮演一个施工图前辈来回答我的问题。

**第三步**：再把 `references/` 里的索引文件发给它。不确定发哪个就**全部发**，AI 会自己判断查哪本。

**第四步**：开始问问题就行了。

> 💡 **小技巧**：
> - `15ZJ001` 这本是"总做法表"，**建议每次都带上**
> - 如果 AI 上下文够长，可以一次性把 SKILL.md + 全部 6 个索引都发进去，效果最好
> - 如果 AI 说"内容太长"，就只发 SKILL.md + 你当前要查的那 1-2 本索引

## 仓库结构

```
.
├── zhongnan-atlas-cd-course/    ← 🎯 skill 本体（安装只需要这个）
│   ├── SKILL.md                 ← AI 行为指令
│   └── references/              ← 6 本图集的文字索引（共 500 KB）
│
├── 原图集/                      ← 📚 6 本 PDF 原件（补充材料，用来核对页码）
│
├── OCR工具/                     ← 🔧 OCR 脚本（想重新生成索引时用）
│
└── README.md                    ← 你正在看的这个
```

**只想用 skill？** → 只需要 `zhongnan-atlas-cd-course/` 文件夹  
**想核对页码？** → 下载 `原图集/` 里的 PDF  
**想重新跑 OCR？** → 用 `OCR工具/` 里的脚本

### references 文件夹说明

`references/` 里的 6 个 `.md` 文件是从扫描版 PDF 用 OCR 识别后整理的结构化索引，包含：
- 每本图集的目录结构
- 每页的关键文字内容（做法编号、材料说明、节点名称等）
- 页码与节点号对照

AI 查询时会读取这些文件，然后告诉用户精确的页码出处。

---

## 准确性说明

- 用 OCR 从无水印版 PDF 识别，平均置信度 **97%**
- 做法编号、规范号、材料名称基本都能准确识别
- **已知局限**：
  - 偶尔有个别错字（不影响查找）
  - **看不到图纸线条**——只能告诉你"去第 X 页看图"，不能描述图形细节
  - PDF 页码和图集印刷页码可能差 1-2 页（封面导致），翻书时前后看一页就能找到

## 常见问题

**Q：AI 说内容太长怎么办？**
只发 SKILL.md + 你当前要查的那 1 本图集索引。优先带 `15ZJ001`（总做法表）。

**Q：AI 给的页码对不上怎么办？**
前后翻 1-2 页。PDF 页码和印刷页码可能有偏差。

**Q：AI 编造了一个做法编号怎么办？**
SKILL.md 里已经要求 AI"找不到就说找不到，不要编造"。如果它还是编了，提醒它："请只引用索引里有的内容，不要编造。"

**Q：我想查的内容索引里没有怎么办？**
说明这个内容在图集里是纯图形（没有文字），OCR 识别不到。这时候只能自己翻 PDF 原件了。

**Q：能不能加更多图集进来？**
可以，但需要跑 OCR 脚本重新生成索引。用 `OCR工具/` 文件夹里的脚本就行。

## 技术细节

- **OCR 引擎**：RapidOCR (onnxruntime)
- **源文件**：无水印版 PDF
- **识别置信度**：平均 97%
- **索引总大小**：~500 KB（原始 PDF 几百 MB → 压缩 99%）
- **Python 依赖**（重新生成索引时需要）：pymupdf, pillow, rapidocr-onnxruntime
