# OCR 工具安装教程(小白版)

> 这份教程教你怎么把 PDF 图集里的文字"识别"出来,让电脑能读。全程大概花你 10 分钟(其中一大半时间是电脑在自己装东西,你只是看着)。

---

## 一、先搞清楚这是在干嘛

你手上的 6 本图集 PDF,里面虽然是"字",但其实是**一张张图片拼起来的**,就像你用手机拍的照片——电脑能看到这张照片,但没法直接认出照片里的字是什么。

**OCR** 这个东西,就是个"电脑识字软件"。装好它,我们就能让电脑把每一页图片里的文字都认出来,变成可以搜索、可以引用的文本。

我们要装的是 **RapidOCR**,它:
- 小巧(整个工具加模型不到 50MB)
- 对中文识别效果好
- 不需要装任何乱七八糟的显卡驱动

---

## 二、装之前确认一件事:你电脑有没有 Python

Python 是运行这个工具的"发动机"。

**怎么查自己有没有装 Python?**

1. 按键盘上的 **Win 键**(屏幕左下角 Windows 图标那个键)
2. 直接敲字 `cmd`,然后按 **回车**
3. 会弹出一个黑底白字的窗口(这就是**命令行**,别怕,它比你想的友好)
4. 在里面敲:
   ```
   python --version
   ```
   按回车

**看到什么就是成功?**
- 如果显示 `Python 3.11.x`(或任何 3.8 以上的版本)→ ✅ 已经装好了,跳到【三、安装 OCR 工具】
- 如果显示 `'python' 不是内部或外部命令` → 需要先装 Python,继续看下一小节

### 如果你没有 Python,装一下

1. 打开浏览器,进入:https://www.python.org/downloads/
2. 点绿色大按钮 **"Download Python 3.x.x"**
3. 下载好的文件双击打开,安装时**一定要勾选最下面那个 "Add Python to PATH"**(非常关键,不勾的话后面命令会用不了)
4. 然后点 "Install Now",等进度条走完
5. 装完后**关掉刚才那个黑色命令行窗口,重新打开一个**(这样新装的 Python 才会生效)
6. 重新输入 `python --version`,能看到版本号就成功了

## 三、安装 OCR 工具(3 个命令搞定)

> ⚠️ 以下所有命令,请在**本项目文件夹**里的命令行运行。
>
> **怎么在本项目文件夹打开命令行?**
> - 打开文件资源管理器,进入到你的图集文件夹(里面有那 6 本 PDF)
> - 在地址栏(就是显示路径的那一长条)**点一下**,把里面内容全选,然后**打 `cmd` 按回车**
> - 会弹出一个黑色窗口,而且开头的路径就是你的图集文件夹 ✅

在这个命令行里,依次敲下面 3 行(每敲一行就按回车,等它跑完再敲下一行):

```
pip install pymupdf
```
这个装 PDF 处理库。大概 10-30 秒。

```
pip install pillow
```
这个装图像处理库。大概 5-10 秒。

```
pip install rapidocr-onnxruntime
```
这个装 OCR 识别引擎,**体积最大**(大概 100MB),可能需要 1-3 分钟。

### 装完怎么验证成功?

每装完一个,命令行最后一行应该出现:
```
Successfully installed xxx-x.x.x
```
看到 `Successfully` 这个词,就是成功了。

### 如果报错怎么办?

**报错 1:"pip 不是内部命令"**
- 说明 Python 安装时没勾 "Add to PATH"
- 解决:重装一遍 Python,**务必**勾上最下面那个选项

**报错 2:下载很慢或失败**
- 这是网络问题,换个镜像源试试:
  ```
  pip install pymupdf -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```
  (后面那一长串是清华大学的镜像,速度飞快)
- 其他两个命令同样加 `-i https://pypi.tuna.tsinghua.edu.cn/simple` 就行

## 四、开始跑 OCR

装好之后,在**同一个命令行窗口**里敲:

```
python OCR工具/ocr识别图集.py
```

### 会发生什么?

**第一次运行(耐心等一下):**
- 会看到 `🔧 正在初始化 OCR 引擎...`
- 它在偷偷下载模型文件(约 10MB),下载好就会继续
- 如果卡在这一步超过 2 分钟,可能是网络问题,看下面【常见问题】

**正常跑起来后:**
- 会看到类似这样的输出:
  ```
  📖 开始处理:15ZJ001《建筑构造用料做法》.pdf
     进度 5/159,已用时 12 秒,预计还需 370 秒
     进度 10/159,已用时 24 秒,预计还需 360 秒
     ...
  ```
- 它会**一本一本**处理,共 6 本,总时间预估 **30-60 分钟**
- 你可以把命令行窗口最小化,去干别的事,电脑会自己跑完

**跑完之后:**
- 最后一行会显示 `🎉 全部完成!`
- 在 `OCR工具/识别结果/` 文件夹里会有 6 个 json 文件
- 这时候跟 Kiro 说一声 "OCR 跑完了",我就会接着帮你建 skill

## 五、常见问题

**Q1:我关掉了命令行,怎么继续?**
- 没事,脚本有"断点续跑"功能。重新打开命令行、再运行 `python OCR工具/ocr识别图集.py`,**已经识别过的图集会自动跳过**,只继续处理没做完的。

**Q2:我想停下来?**
- 按 `Ctrl + C` 可以随时中断。下次继续跑时,已经做完的图集不会重复。

**Q3:识别结果可以用来干什么?**
- 我(Kiro)会用它来建立 skill 的索引,这样以后你问"屋面保温层应该用哪种材料"的时候,我能**准确告诉你在哪本图集的哪一页**。

**Q4:初始化卡住下载不下来?**
- 临时用手机热点或换个网络试试
- 或者手动下载模型(需要了再找我)

**Q5:识别结果准吗?**
- RapidOCR 对印刷体中文的识别率在 95% 以上
- 图集里的做法编号、页码、章节标题基本都能准确识别
- 少量小字或模糊地方可能会错,但不影响你通过关键字找到对应页面
